Capabilities

Everything you need, nothing you don't

Built with browser-native APIs — no plugins, no installs, no accounts. Just open and transfer.

🔐

Military-grade Encryption

Every chunk encrypted with AES-256-GCM before leaving your device. ECDH P-256 key exchange, HKDF-SHA-256 derivation, SHA-256 integrity verification.

⚡

True Peer-to-Peer

WebRTC DataChannel creates a direct device-to-device connection. The signaling server only facilitates the handshake — it never sees your files.

📤

Drag & Drop Transfer

Drop files onto the zone or directly onto a peer card. Click to browse or press ⌘O. Any file type, any size — limited only by your device RAM.

📱

QR Code Join

Scan the in-app QR code to instantly connect a phone or tablet. The URL encodes the signaling server address — zero manual setup on the joining device.

🌐

Platform Independent

Runs in any modern browser — Chrome, Safari, Firefox, Edge. Works on Mac, Windows, Linux, iOS, and Android. No native app required.

🛡

MITM Detection

Both peers derive a session fingerprint from the ECDH shared secret. Read it aloud to confirm no man-in-the-middle is intercepting the connection.

🌙

Dark & Light Mode

Sci-fi dark mode by default. Toggle to a clean light mode with one click. Theme preference is persisted across sessions.

📊

Real-time Monitoring

Live transfer speed, progress bars, CPU usage, memory, network throughput, and uptime — all visible in the system dashboard.

🔁

Auto-Reconnect

Signaling connection drops are handled with exponential backoff (1s → 30s). Peers re-appear automatically after reconnection.

Architecture

The server is just a matchmaker

The signaling server handles only tiny JSON handshake messages. All file data flows directly between devices.

SENDER (Browser A)                                  RECEIVER (Browser B)
┌───────────────────────────────────┐                ┌──────────────────────────────────┐
│  SignalingClient  ─── WebSocket ──┼────────────────┼── WebSocket ──  SignalingClient  │
│  PeerConnection   ─── ICE/SDP ───┼────────────────┼── ICE/SDP ───   PeerConnection  │
│  CryptoService    (ECDH + HKDF)  │                │  (ECDH + HKDF)   CryptoService  │
│  ChunkTransfer  ──── DataChannel ┼════════════════┼═══ DataChannel ─  ChunkTransfer  │
│  IntegrityVerify                 │   Direct P2P   │                  IntegrityVerify │
└───────────────────────────────────┘  (no server)  └──────────────────────────────────┘
              │                                                   │
              └──────────────┐  Signaling Server  ┌──────────────┘
                             │  ─────────────────  │
                             │  Python WebSocket   │
                             │  (peer registry +   │
                             │   SDP/ICE relay)    │
                             │  NEVER sees files   │
                             └─────────────────────┘

Data path:   File → Encrypt(AES-256-GCM) → Chunk(16KB) → WebRTC P2P → Decrypt → File
Crypto:      ECDH P-256 → shared secret → HKDF-SHA-256 → AES-256 key → per-chunk IV

Security

Cryptography you can verify

Every security property is implemented using the browser's built-in Web Crypto API — auditable, standard, and hardware-accelerated.

Property Standard Implementation
Key exchange NIST FIPS 186-4 ECDH P-256 — ephemeral per session, no pre-shared secrets
Key derivation RFC 5869 HKDF-SHA-256 — context-bound to "secure-drop-v1"
Encryption NIST SP 800-38D AES-256-GCM — per-chunk, unique 96-bit IV, 128-bit auth tag
Integrity FIPS 180-4 SHA-256 of complete file, verified on receipt before saving
MITM detection Custom 8-char hex fingerprint from SHA-256(raw ECDH shared bits) — read aloud to verify
Forward secrecy — Ephemeral ECDH keys — compromise of one session reveals nothing about others
Server trust — Signaling server sees only peer IDs + SDP — never keys or file data
Get Started

Up and running in minutes

📡 Same Wi-Fi (LAN)

Zero config — just run the launcher script. Works offline.

pip3 install websockets zeroconf ./start-lan.sh # Opens on: # This device → localhost:8080 # Other devices → 192.168.X.X:8080 # Or scan QR code in the app

🌐 Internet (Anywhere)

Deploy once — works from any device, any network, free.

# 1. Deploy signaling → Railway # Root dir: server # Gets URL: wss://your.railway.app # 2. Deploy frontend → Netlify # Env var: VITE_SIGNALING_URL=wss://your.railway.app # Build: npm run build # Publish: dist # 3. Open Netlify URL on any device
